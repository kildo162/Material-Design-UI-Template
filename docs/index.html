Import project in Android Studio - top

Android Studio project is in folder "android - AS". First, open AS, click File -> Close Project...then select "Open an Existing Android Studio project". After that, find downloaded folder from codecanyon, and find folder "android - AS".

Wait few moments for studio to finish synchronizing and building and after that you are free to run UIAppTemplate project

Structure of This App Template

This demo app's structure is: you have MainActivity which basicly implements native Android left menu (Navigation Drawer). When you open the app, you'll see that front page is just first item selected from left menu. In MainActivity you have 12 left menu items (more will come in future). Left menu is just a ListView, so left menu items are just objects of DrawerItem class added in DrawerAdapter class. DrawerAdapter is used to populate left menu ListView. DrawerItem class has three fields: icon, title and tag. Icon and title fields are integer values that represent Resource values of partucular icon and title related to that item in left menu. Tag is one of the DrawerModel's public, static and final String fields that start with DRAWER_ITEM_TAG_. This tag is used to determine which item from left menu list is clicked. This way, you can reorder items easily since clicking on item is not related to position but to tag. Clicking the item replaces Fragment in FrameLayout that is defined in main_activity.xml. Every fragment is defined in ~.fragment package. To see what fragment is used for what, you can open any fragment. We gave names to fragments that are equal to the item they represent ending with ~Fragment.java. Some fragments only have ListView as a main content (and items in list open new Activity that shows real content), and others have real content right away.

Roboto Text View

We use tehnique of defining typeface through xml (click here for more details). Idea is to create a new class that inherits TextView. Then we add custom defined attribute to attrs.xml file by declaring styleable RobotoTextView. Then, in related construtors of RobotoTextView class we read if there were defined attribute called typeface. If it exists, we set typeface according to the value of the attribute. Here is snippet of code:

<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"     xmlns:app="http://schemas.android.com/apk/res/com.csform.android.uiapptemplate"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content">

    <com.csform.android.uiapptemplate.font.RobotoTextView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        app:typeface="robotoBold" />

</LinearLayout>

To use custom defined attribute in an xml layout file, you need to add namespace like we did with namespace app. You can set any name that you want (we decided to use app). If you however want to define attribute value through styles, all you need to do is to use attribute name without namespace prefix (take a look at how we set typeface attribute in styles_text_view.xml file.)

IMPORTANT: If you want to use this class to set typeface through xml in your project(s), copy folder fonts that is located in folder assets. Also, you should create attrs.xml file (if you haven't) and create styleable RobotoTextView like we did in our attrs.xml file.

Material Design Icons (Fontello) text view

Another tehnique we use in this app is a tehnique of using text with appropriate font instead of icons. How do we achieve this? We use fontello.com generator to generate font file. It generates few font file extensions, but we use .ttf extension. Then, when fontello generates fonts, you get zip file with codes for each character/icon. When you open codes, you will see that they are written in hexadecimal format. Then we use that format to set them in strings.xml.

Then, all you need to do is to set text defined in strings.xml resource to Fontello text view. And you get icon that is actually a font defined for set of characters fontello.com has given to us. This way, we avoided using png icons in (at least) four dimensions (in x-dpi folders) thus decreased size of the app.

You can see example of using this tehnique in res/layout/list_view_item_navigation_drawer_1.xml xml file or in res/layout/activity_login_page_dark.xml.

NOTE: Whenever we use this tehnique to set icon, we set textSize in dps. You should never set text size in dps unless you have a good reason. And we have one: we don't want our icons grow big when user enlarges entire phone text size through global settings.

IMPORTANT: To use our or custom set of fontello icons, copy/paste MMaterialDesignIcons.ttf file (from our project) to assets/fonts folder.

This demo project is called UIAppTemplate in workspace and you can run it and test from Eclipse/Android Studio. Every project from GitHub is imported into workspace and converted to library project, but only one is imported as library project into our demo project (StickyListHeaders). So, if you would like to use features from these GitHub projects, you can import them as library projects into your own (to do so, please take a look at this page).

Since every project taken from GitHub is not imported as a library, but just copy/pasted into our UIAppTemplate project, you may have some issues implementing our UI items that use foreign projects components. Eg. if you want to use ProgressWheel in your project, you have two options. First is to import ProgressWheel from our workspace and then import that project as library into your project. Second option is to copy/paste our file ProjectWheel.java (which is basicly copy/pasted file from GitHub project) into you project (as we did). If you choose second option, you might stuck with errors, usually something with generated R file. If you get with R cannot be resolved to a variable error, organize your imports and import R from your project (it has the same package name as your project). Then, if you come with R.styleable.Something cannot be resolved to a variable, open attrs.xml file from our project and find appropriate declare-styleable, then copy/paste it to you project's attrs.xml file. If you come with R.raw cannot be resolved to a variable, copy/paste raw folder. And so on. If you stuck with an error you can not solve, please contact us and we will try to help you.

Image Gallery - top

An example of image gallery elements. Very important tehnique we used here is to use getItemViewType(int) of a BaseAdapter. For more information, take a look at here.

So, why did we use ListView if gallery looks like it was used GridView. First, we needed a header to show title of a Category (to add header view, please take a look at Vogella article). To set header view, we had to use ListView. GridView does not support adding header views - logically. So, we had to create list items that look like grid items when GridView is set to 2 columns. So, we had to have items in ListView row that has both one and two images. We always have two images in a row except last row if there is odd number of items. So, instead of switching between View.GONE and View.VISIBLE to the last item, we created two list item xmls that do this for us. And then, in adapter, we checked for a type of item that is returned from method getItemViewType(position), then we choose which xml to inflate. Xmls we used here are: list_item_categories_two_columns.xml and list_item_categories_one_column.xml.

If you use your own Activity, use AppTheme.NoShadow style as defined in Mainfest for ImageGalleryCategoryActivity and ImageGallerySubcategoryActivity.

NOTE: Category item layout has two different positions to show caption text and favourite image. These positions are on top or bottom of the image. To define position, we have to set boolean value in ImageGalleryCategoryAdapter or ImageGallerySubcategoryAdapter constructor. False value means that the layout that contains caption text and favourite image is positioned at the bottom of the parent (parent is RelativeLayout that contains ImageView and the layout). True means that the layout is positioned at the top of the parent. All these is reated to list_item_categories_two_columns.xml and list_item_categories_one_column.xml layouts and @+id/layout_top_bottom_1 and @+id/layout_top_bottom_2 ids that are defined for LinearLayout in each of the two layouts.

Left Menus - top

Navigation drawer layout, most commonly known as a left menu is just another ListView that is placed into a special layout (android.support.v4.widget.DrawerLayout) capable of showing when swipped from far left position of the screen. To implement native Android left menu, please visit this link.

So, if a left menu is just a ListView, there is nothing special to add here except that we added header view by inflating layout/header_navigation_drawer_1 or layout/header_navigation_drawer_2 xmls.

Only thing you should be warned about is that we have two xmls that define item in list. First one has icon, but second one does not have it. So, creating adapter object (of DrawerAdapter class), we had to pass boolean value to indicate what layout to inflate. True is for layout/header_navigation_drawer_1, false is for layout/header_navigation_drawer_2.

NOTE: If you are about to add custom header view to ListView, do it before calling setAdapter(Adapter) method of a ListView. This is important if you are supporting pre-KitKat devices. We did so, so there is nothing you should be worried about for now.

Firebase notifications - top

Firabase notifications are implemented in project. You will need to replace dummy file google-services.json inside UIAppTemplate folder with your own to enable them.

In order to do that you will need Firebase account. In Firebase console create new project. After that click on Add Firebase to you Android app. For the package name enter com.csform.android.uiaptemplate or your package name if you already change it (and you should). Clicking on ADD APP button you will be provided with an option to download your own google-services.json file.

Subscribe on Mailchimp list - top

Project now shows subscribe dialog after you click 3 times on main menu. You can change that number in MainActivity.java file SUBSCRIBE_VIEW field. To be able to subscribe to your own list you must provide mailchimp api-key and list id into res/values/strings.xml. Put your list id inside mailchimp_list_id. Api key is in format xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx-xxx. Put xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx part of the key inside mailchimp_key and xxx part into server_num
